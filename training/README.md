# Ask Physics QLoRA Data Project

This subproject builds the open-source data pipeline needed to fine-tune a small open model for Ask Physics with QLoRA.

The first goal is not model training. The first goal is a reproducible dataset:

```text
arXiv metadata and abstracts
  -> weak regex labels
  -> reviewed/teacher labels later
  -> model-neutral SFT JSONL
  -> Gemma or Mistral QLoRA training
```

## Scope

The dataset targets two Ask Physics tasks:

1. Abstract extraction: extract plasma physics measurements from title and abstract text.
2. Query parsing: convert user search queries into structured JSON for SPARQL generation.

This first project implements the abstract extraction path. Query parsing examples should be added in a later phase after the extraction dataset format is stable.

## Data Levels

- `raw`: arXiv metadata records with title, abstract, authors, categories, dates, and source URLs.
- `bronze`: weak labels generated by deterministic regex extraction.
- `silver`: labels generated by regex plus a teacher model, after schema validation.
- `gold`: manually reviewed labels used for evaluation only.
- `sft`: chat/instruction JSONL used for Gemma or Mistral QLoRA supervised fine-tuning.

## Quick Start

Install dependencies from the repo root:

```bash
python3 -m pip install -r training/requirements.txt
```

Collect a small raw dataset:

```bash
python3 training/scripts/collect_arxiv_metadata.py \
  --max-results 25 \
  --output training/data/raw/arxiv_metadata.jsonl
```

Build bronze labels with the repo's regex extractor:

```bash
python3 training/scripts/build_bronze_labels.py \
  --input training/data/raw/arxiv_metadata.jsonl \
  --output training/data/bronze/regex_labels.jsonl
```

Prepare model-neutral SFT data:

```bash
python3 training/scripts/prepare_sft_dataset.py \
  --labels training/data/bronze/regex_labels.jsonl \
  --output-dir training/data/sft
```

## Gemma QLoRA Training

The runnable Gemma QLoRA package is:

```text
training/configs/gemma_qlora.yaml
training/scripts/train_gemma_qlora.py
training/scripts/format_sft_for_gemma.py
training/scripts/run_gemma_inference.py
training/scripts/evaluate_gemma_extraction.py
training/run_gemma_cuda.sh
training/GEMMA_QLORA_RUNBOOK.md
```

Run this on a CUDA-compatible PyTorch GPU runtime. Gemma model access may require accepting the model terms on Hugging Face and authenticating with `huggingface-cli login` or `huggingface_hub.login()`.

Install CUDA PyTorch for your machine first, then:

```bash
python -m pip install -r training/requirements-qlora.txt
```

For the first training run, use the balanced split:

```text
training/data/sft_balanced/train.jsonl
training/data/sft_balanced/dev.jsonl
training/data/sft_balanced/test.jsonl
```

Start training:

```bash
bash training/run_gemma_cuda.sh
```

For the full command-line runbook:

```text
training/GEMMA_QLORA_RUNBOOK.md
```

After training, run a smoke inference:

```bash
python training/scripts/run_gemma_inference.py \
  --config training/configs/gemma_qlora.yaml \
  --adapter-dir training/runs/gemma-qlora-ask-physics/adapter \
  --title "Example plasma diagnostic paper" \
  --abstract "The electron temperature reached 5.2 keV and the electron density was 7.1 x 10^19 m^-3 during the discharge."
```

Evaluate JSON and coarse extraction quality:

```bash
python training/scripts/evaluate_gemma_extraction.py \
  --config training/configs/gemma_qlora.yaml \
  --adapter-dir training/runs/gemma-qlora-ask-physics/adapter \
  --eval-file training/data/sft_balanced/test.jsonl \
  --limit 50
```

## Model Formatting

The SFT output uses a generic `messages` JSONL structure:

```json
{"messages":[{"role":"system","content":"..."},{"role":"user","content":"..."},{"role":"assistant","content":"..."}]}
```

Keep this dataset model-neutral. Apply the Gemma or Mistral tokenizer chat template during training, not during data collection.

## Open-Source Rules

- Prefer arXiv metadata and abstracts for v0.1.
- Keep source URLs and arXiv IDs in every artifact.
- Track label provenance: `regex`, `teacher`, or `human`.
- Do not redistribute full PDFs or full text unless the source license allows it.
- Split train/dev/test by `paper_id`, not by row.
